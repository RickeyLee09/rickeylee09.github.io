HOW TO USE
GIT

This page shows some basic usages of GIT.
Basically, including common command lines and functionality of each of them

Git: git diff

比较当前工作目录及暂存区的区别,可以指定分支及文件. 如果还未add进暂存区,则比较文件自身修改前后的区别
                        $ git diff
                        $ git diff 'filename'
                        $ git diff 'branch' 'filename'
比较远程仓库两次commit的差异
                        $ git diff 'commit' 'filename'
                        $ git diff 'commit' 'commit'

Git: git branch

                        $ git branch                //查看当前分支
                        $ git branch -r             //查看远程仓库当前分支
                        $ git branch -d 'branch'    //删除指定本地分支
                        $ git branch -r -d 'branch' //删除指定远程仓库分支

                        $ git checkout 'brunch'     //切换分支
                        $ git checkout -b 'brunch'  //创建并切换分支

                        $ git merge 'brunch'        //合并分支

Git: 上传常用操作

本地仓库暂存后提交到远程仓库流程
                        // 本地仓库操作
                        $ git init                              //初始化一个新仓库
                        $ git add                               //添加文件到暂存区
                        $ git commit -m "commit instruction"    //提交更新

                        // 远程仓库操作
                        $ git remote add 'shortname' 'url'      //添加远程仓库
                        $ git push 'shortname' 'brunch'         // 上传至远程仓库

Git: 拉取远程仓库文件

git pull vs git fetch

git fetch：相当于是从远程获取最新版本到本地，不会自动merge
git pull：相当于是从远程获取最新版本并merge到本地
在实际使用中，git fetch更安全一些,因为在merge前，我们可以查看更新情况，然后再决定是否合并
